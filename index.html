HTML

  • HTML acts as a skeleton for our webpages as it is language we use to structure out websites
  • Many elements have class/id attributes that can give info about or even change the behavior/look of our elements
  • The head element contains information about the webpage.
  • Within the head we have common child elements like <'title'> & <'meta'>
  • The body element represents the visible content shown to the user
  • Inside our body element we have many child elements we can use but headings like <'h1'> and paragraph elements (<'p'>) are most common
  • Non descriptive (a.k.a non-semantic) elements like <'div'> and <'span'> have their place but semantic elements like <'form'> or <'table'> are preffered

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: Takes the most updated version of our project from the main remote repository and updates our local main branch
  • git add -A: adds all the files in our working directory to the staging area to be commited
  • git commit -m "commit-message": creates a version of our code that can be passed up to the main branch or at the very least used as a reference point to go back to. The commit message gives a brief description of the change in the commit
